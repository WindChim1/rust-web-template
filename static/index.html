核心技术栈与特性

Salvo & Tokio

基于最新的 Salvo 框架，利用 Rust 强大的异步运行时 Tokio，提供极高的并发处理能力和内存安全性。

SQLx & Postgres

纯 Rust 编写的异步 SQL 库，支持编译时 SQL 检查。内置 Postgres 连接池管理，安全且高效。

AOP 操作日志

基于 Salvo Middleware 实现的无侵入式日志系统。自动捕获请求参数、响应结果、耗时及操作人信息并入库。

多环境配置

支持 dev.toml, release.toml 等多配置文件，通过环境变量动态切换，完美适配 Docker 部署。

Workspace 架构

采用 Rust Workspace 管理多包项目，将 Framework 与 Modules 物理隔离，依赖清晰，编译更快。

Docker 友好

提供优化的 Dockerfile，支持交叉编译 (Cross)，支持挂载配置文件和可执行文件，实现秒级热更新。

模块化项目结构

采用 Rust Workspace 模式，实现“核心框架”与“业务模块”的物理分离。

  • app/ // 启动入口 (main.rs, lib.rs)
  • common/ // 公共工具库
  • config/ // 环境配置
    • dev.toml (开发环境)
    • release.toml (生产环境)
    • test.toml
  • framework/ // 基础设施底座
    • config.rs (配置加载)
    • db.rs (SQLx 连接池)
    • jwt.rs (Token 认证)
    • log.rs (日志封装)
    • middleware.rs (全局中间件)
  • modules/ // 业务领域层
    • monitor/ // 系统监控
      login_info, operlog
    • system/ // 系统管理
      user, role, menu, dict, file

分层设计理念

1

Framework (核心层)

位于 framework 目录。封装了与业务无关的基础能力，如 DB 连接、鉴权、日志、配置解析。它作为 crate 被其他模块引用。

2

Modules (业务层)

位于 modules 目录。按领域驱动设计 (DDD) 拆分为 system (用户权限) 和 monitor (监控审计) 等独立 crate。每个模块包含自己的 handler、service 和 model。

3

App (组装层)

位于 app 目录。作为二进制入口，负责读取配置、初始化 Framework、挂载各个 Module 的路由，最终启动 Salvo Server。

快速开始

1. 克隆模板与配置

cargo generate WindChim1/rust-web-template

cd {project_name}
# 复制配置文件
cp config/dev.toml.example config/dev.toml

2. 运行开发环境

# 确保本地已启动 PostgreSQL
# 在 Workspace 根目录运行
cargo run -p app

3. 业务代码示例 (Handler)

编写业务接口极其简洁，无需关心日志细节：

#[handler]
pub async fn add_article(req: &mut Request, depot: &mut Depot, res: &mut Response) -> AppResult<()> {
    // 1. 获取 DB 链接
    let pool = depot.obtain::().unwrap();
    // 2. 解析参数
    let dto = req.parse_json::().await?;
    // 3. 执行 Service
    let id = service::add(pool, dto).await?;

    // 4. 【关键】标记日志（一行代码实现审计）
    LogMeta::set(depot, "用户管理", BusinessType::Add.get_value(), "添加用户");

    ResponseResult::success_msg("添加成功").into()

}

Docker 部署方案

方案：挂载式更新 (推荐)

使用基础运行时镜像，通过挂载宿主机可执行文件实现秒级更新，无需重复构建镜像。

# 启动容器
docker run -d \
  --name app-server \
  -p 8080:8080 \
  -e RUN_MODE=release \
  -v $(pwd)/config:/app/config \
  -v $(pwd)/logs:/app/logs \
  -v $(pwd)/uploads:/app/uploads \
  -v $(pwd)/target/release/app-linux:/app/server \
  rust-runtime:v1

更新流程

  1. 本地使用 cross 编译 Linux 二进制文件。
  2. 停止容器：docker stop app-server。
  3. 替换宿主机的 app-linux 文件。
  4. 启动容器：docker start app-server。

⚠️ 注意：在 Windows 环境下开发，务必使用 cross build --target x86_64-unknown-linux-gnu 进行交叉编译。
